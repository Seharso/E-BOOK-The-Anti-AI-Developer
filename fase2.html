Fase 2: Seni Membangun Antarmuka

Durasi: 6-8 Minggu

Tujuan: Mampu membuat website yang interaktif, responsif, dan enak dipandang.

Materi dalam Fase Ini

1. HTML5: Kerangka Website

Penjelasan Detail

Semantic HTML: <header>, <nav>, <main>, <section>, <article>, <footer>. Form, input types, media (img, video).

Proyek Nyata

Proyek 4: Halaman Profil Statis. Buat halaman HTML sederhana tentang Anda. Gunakan semantic HTML. Include foto, paragraf bio, daftar skill, dan form kontak (tanpa styling dulu).

Tips & Trik

Gunakan ekstensi "Emmet" di VS Code untuk menulis HTML lebih cepat.

Warning

Jangan gunakan <table> untuk layout. Itu adalah praktik kuno.

Best Practice

Selalu sertakan atribut alt pada tag <img> untuk aksesibilitas. Validasi HTML Anda menggunakan W3C Validator.

Prompt AI Siap Pakai
[PERAN AI] Act as a web accessibility specialist. [TUJUAN] I want to build a website that is accessible to everyone. [INSTRUKSI] Explain the importance of semantic HTML5 tags for accessibility and SEO. Give me 5 common accessibility mistakes beginners make with HTML and how to fix them.

2. CSS3: Gaya & Tata Letak

Penjelasan Detail

Selectors, Box Model, Positioning, Flexbox, CSS Grid. Transisi & animasi dasar. Konsep responsive design dengan media queries.

Proyek Nyata

Proyek 5: Halaman Profil Responsif. Ambil Proyek 4 dan berikan styling dengan CSS. Buat layout yang rapi dengan Flexbox/Grid. Pastikan tampilan bagus di desktop dan mobile (gunakan media queries).

Tips & Trik

Gunakan Chrome DevTools (F12) untuk meng-inspect dan bereksperimen dengan CSS secara real-time.

Warning

"Div soup" (terlalu banyak <div> tanpa makna) adalah tanda HTML yang buruk.

Best Practice

Gunakan Custom Properties (CSS Variables) untuk warna dan font yang berulang. Ini memudahkan maintenance.

Prompt AI Siap Pakai
[PERAN AI] Act as a senior CSS developer. [TUJUAN] I'm confused about when to use Flexbox and when to use CSS Grid. [INSTRUKSI] Explain the primary use cases for both, their core differences (1D vs 2D layout), and provide a visual example (using ASCII art or a clear description) of a layout that is better suited for Grid and one for Flexbox.

3. JavaScript DOM: Membuat Halaman Interaktif

Penjelasan Detail

Mengakses elemen DOM (querySelector, getElementById). Manipulasi elemen (ubah teks, style, atribut). Event handling (click, submit, scroll). Web APIs: fetch untuk mengambil data dari server.

Proyek Nyata

Proyek 6: Aplikasi Cuaca Sederhana. Buat halaman dengan input kota. Saat user submit, gunakan fetch untuk mengambil data cuaca dari API publik (misal: OpenWeatherMap) dan tampilkan hasilnya di halaman.

Tips & Trik

Delegasi event adalah teknik powerful untuk menangani event di banyak elemen secara efisien.

Warning

Jangan memblokir rendering dengan JavaScript yang lambat. Letakkan tag <script> di akhir <body> atau gunakan defer.

Best Practice

Pisahkan concerns: HTML untuk struktur, CSS untuk style, JavaScript untuk logika. Jangan gunakan inline onclick atribut di HTML.

Prompt AI Siap Pakai
[PERAN AI] Act as a frontend mentor. [TUJUAN] I'm trying to build a to-do list app but my 'delete' button doesn't work on new items I add. [INSTRUKSI] Explain the concept of "Event Delegation" in the DOM. Show me a 'before' code example that has this problem and an 'after' code example that uses event delegation to solve it.

Proyek Nyata

Proyek 4: Halaman Profil Statis

Buat halaman HTML sederhana tentang Anda. Gunakan semantic HTML. Include foto, paragraf bio, daftar skill, dan form kontak (tanpa styling dulu).

Proyek 5: Halaman Profil Responsif

Ambil Proyek 4 dan berikan styling dengan CSS. Buat layout yang rapi dengan Flexbox/Grid. Pastikan tampilan bagus di desktop dan mobile (gunakan media queries).

Best Practices & Tips

Best Practices untuk Fase 2:

  • Selalu sertakan atribut alt pada tag <img> untuk aksesibilitas.
  • Validasi HTML Anda menggunakan W3C Validator.
  • Gunakan ekstensi "Emmet" di VS Code untuk menulis HTML lebih cepat.
  • Gunakan Chrome DevTools (F12) untuk meng-inspect dan bereksperimen dengan CSS secara real-time.
  • Gunakan Custom Properties (CSS Variables) untuk warna dan font yang berulang.
  • Delegasi event adalah teknik powerful untuk menangani event di banyak elemen secara efisien.
  • Pisahkan concerns: HTML untuk struktur, CSS untuk style, JavaScript untuk logika.
  • Jangan memblokir rendering dengan JavaScript yang lambat.
