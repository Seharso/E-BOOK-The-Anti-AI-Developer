Fase 3: Otak & Jantung Aplikasi

Durasi: 8-10 Minggu

Tujuan: Mampu membuat server, mengelola data, dan membangun API yang akan digunakan oleh frontend.

Materi dalam Fase Ini

1. Node.js & Express.js

Penjelasan Detail

Konsep server, request/response cycle. Menginstall Express, routing, middleware. Menangani parameter di URL dan request body.

Proyek Nyata

Proyek 7: API To-Do List. Buat server dengan Express. Buat endpoint: GET /todos (ambil semua), GET /todos/:id (ambil satu), POST /todos (tambah baru), DELETE /todos/:id (hapus). Untuk sementara, simpan datanya di array di memory server.

Tips & Trik

Gunakan nodemon agar server restart otomatis saat ada perubahan kode.

Warning

Jangan pernah mempercayai input dari user (client). Selalu lakukan validasi di server.

Best Practice

Struktur folder yang baik: routes, controllers, models, middleware.

Prompt AI Siap Pakai
[PERAN AI] Act as a Node.js/Express expert. [TUJUAN] I don't understand what middleware is. [INSTRUKSI] Explain middleware using the analogy of an assembly line or a series of security checkpoints. Show me how to write a simple logging middleware and an error-handling middleware in Express.

2. Database: SQL & PostgreSQL

Penjelasan Detail

Konsep database relasional. Query dasar: CREATE TABLE, INSERT, SELECT, UPDATE, DELETE. JOIN. Menghubungkan Node.js ke database dengan library seperti pg atau ORM seperti Prisma/Sequelize.

Proyek Nyata

Proyek 8: API To-Do List dengan Database. Modifikasi Proyek 7. Ganti array di memory dengan database PostgreSQL. Buat tabel todos. Semua operasi CRUD sekarang harus berinteraksi dengan database.

Tips & Trik

Mulai dengan SQL murni sebelum beralih ke ORM. Ini membantu Anda memahami apa yang terjadi di balik layar.

Warning

SQL Injection adalah serangan berbahaya. Gunakan prepared statements atau parameterized query (ORM melakukan ini untuk Anda).

Best Practice

Gunakan environment variables (.env file) untuk menyimpan kredensial database. Jangan hardcode di dalam aplikasi.

Prompt AI Siap Pakai
[PERAN AI] Act as a database administrator. [TUJUAN] I need to design a database for a simple blog. [INSTRUKSI] Propose a database schema for a blog with 'users', 'posts', and 'comments'. Show me the SQL CREATE TABLE statements for each table and explain the relationships (one-to-many, etc.) between them using foreign keys.

3. Autentikasi & Keamanan

Penjelasan Detail

Konsep JWT (JSON Web Tokens). Hashing password (dengan bcrypt). Membuat sistem register & login. Middleware untuk melindungi route (hanya user yang login bisa akses).

Proyek Nyata

Proyek 9: API dengan Register & Login. Tambahkan fitur user pada API To-Do List. User harus register dan login untuk mendapatkan token. Token ini digunakan untuk mengakses endpoint todos miliknya saja.

Tips & Trik

Simpan JWT di HttpOnly cookie, bukan di localStorage untuk mencegah serangan XSS.

Warning

Jangan pernah menyimpan password dalam bentuk plain text. SELALU hash.

Best Practice

Prinsip terkecil hak istimewa (Principle of Least Privilege). Berikan akses seminimal mungkin.

Prompt AI Siap Pakai
[PERAN AI] Act as a cybersecurity expert. [TUJUAN] I want to implement user authentication. [INSTRUKSI] Explain the flow of JWT-based authentication from user login to accessing a protected route, step-by-step. Also, explain what 'hashing' a password is, why it's crucial, and how it's different from encryption.

Proyek Nyata

Proyek 7: API To-Do List

Buat server dengan Express. Buat endpoint: GET /todos (ambil semua), GET /todos/:id (ambil satu), POST /todos (tambah baru), DELETE /todos/:id (hapus). Untuk sementara, simpan datanya di array di memory server.

Proyek 8: API To-Do List dengan Database

Modifikasi Proyek 7. Ganti array di memory dengan database PostgreSQL. Buat tabel todos. Semua operasi CRUD sekarang harus berinteraksi dengan database.

Proyek 9: API dengan Register & Login

Tambahkan fitur user pada API To-Do List. User harus register dan login untuk mendapatkan token. Token ini digunakan untuk mengakses endpoint todos miliknya saja.

Best Practices & Tips

Best Practices untuk Fase 3:

  • Gunakan nodemon agar server restart otomatis saat ada perubahan kode.
  • Struktur folder yang baik: routes, controllers, models, middleware.
  • Jangan pernah mempercayai input dari user (client). Selalu lakukan validasi di server.
  • Mulai dengan SQL murni sebelum beralih ke ORM. Ini membantu Anda memahami apa yang terjadi di balik layar.
  • Gunakan environment variables (.env file) untuk menyimpan kredensial database.
  • Simpan JWT di HttpOnly cookie, bukan di localStorage untuk mencegah serangan XSS.
  • SELALU hash password, jangan pernah menyimpan dalam bentuk plain text.
  • Terapkan prinsip terkecil hak istimewa (Principle of Least Privilege).
