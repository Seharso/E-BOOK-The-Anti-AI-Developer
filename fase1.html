Fase 1: Bahasa Pemrograman & Logika Fundamental

Durasi: 4-6 Minggu

Tujuan: Menguasai logika dan sintaks dasar pemrograman. Kami memilih JavaScript karena sangat populer, versatile (bisa untuk frontend, backend, mobile), dan memiliki komunitas besar.

Materi dalam Fase Ini

1. Variabel & Tipe Data

Penjelasan Detail

let, const, var. Tipe data: String, Number, Boolean, Null, Undefined, Object, Array.

Proyek Nyata

-

Tips & Trik

Gunakan const secara default, let hanya jika nilai variabel akan berubah. Hindari var.

Warning

Jangan lupa bahwa JavaScript memiliki type coercion yang bisa tricky. 5 + "5" hasilnya "55".

Best Practice

Beri nama variabel yang deskriptif. userName lebih baik dari x.

Prompt AI Siap Pakai
[PERAN AI] Act as a JavaScript fundamentals tutor. [TUJUAN] I need to understand variable declarations. [INSTRUKSI] Explain the differences between var, let, and const with clear code examples. Specifically, focus on scope, hoisting, and reassignment. Create a small quiz with 3 questions to test my understanding.

2. Operator & Kontrol Alur (Control Flow)

Penjelasan Detail

Operator aritmatika, perbandingan, logika. if/else, switch, for loop, while loop.

Proyek Nyata

Proyek 1: Kalkulator Sederhana (Console). Buat program yang meminta 2 angka dan operator (+, -, *, /), lalu tampilkan hasilnya.

Tips & Trik

Gunakan === (strict equality) daripada == untuk menghindari bug tak terduga.

Warning

Hati-hati dengan infinite loops. Pastikan kondisi loop Anda akan terpenuhi untuk berhenti.

Best Practice

Untuk iterasi array, gunakan forEach atau map (nanti akan dipelajari) daripada for loop klasik, lebih modern dan mudah dibaca.

Prompt AI Siap Pakai
[PERAN AI] Act as a coding interviewer. [TUJUAN] I want to practice control flow. [INSTRUKSI] Give me a classic beginner problem: "Write a function that takes a number and returns 'Fizz' if it's divisible by 3, 'Buzz' if it's divisible by 5, and 'FizzBuzz' if it's divisible by both. Otherwise, return the number." After I give you my answer, review it and suggest improvements.

3. Fungsi (Functions)

Penjelasan Detail

Deklarasi fungsi, ekspresi fungsi, arrow function. Konsep parameter, argumen, return. Scope (global vs lokal).

Proyek Nyata

Proyek 2: Konversi Suhu. Buat fungsi celsiusToFahrenheit(celsius) dan fahrenheitToCelsius(fahrenheit). Panggil fungsi ini dengan beberapa nilai dan tampilkan hasilnya di console.

Tips & Trik

Fungsi harus melakukan satu hal dengan baik (Single Responsibility Principle).

Warning

Jangan membuat fungsi dengan terlalu banyak parameter (lebih dari 3-4). Jika perlu, gunakan objek sebagai parameter.

Best Practice

Gunakan default parameter untuk membuat fungsi lebih robust. function greet(name = 'Guest') { ... }

Prompt AI Siap Pakai
[PERAN AI] Act as a code quality expert. [TUJUAN] I just wrote a function that is 50 lines long and does many things. [INSTRUKSI] Explain the concept of "Single Responsibility Principle" for functions. Show me how to refactor a hypothetical long function into smaller, more manageable functions. Provide a "before" and "after" code example.

4. Struktur Data: Array & Object

Penjelasan Detail

Manipulasi array: push, pop, shift, unshift, slice, splice. Method tingkat tinggi: map, filter, reduce, find. Manipulasi object: add, delete, access properties.

Proyek Nyata

Proyek 3: To-Do List (Console). Simpan to-do items dalam sebuah array of objects. Setiap object memiliki id, text, dan completed (boolean). Buat fungsi untuk addTodo, deleteTodo, dan toggleComplete.

Tips & Trik

Method map, filter, reduce adalah pilar pemrograman fungsional di JavaScript. Kuasai ini!

Warning

Jangan mengubah array secara langsung (mutation). Gunakan map atau filter yang mengembalikan array baru. Ini mencegah side effects.

Best Practice

Gunakan console.log() secara liberal untuk debugging. Lihat isi variabel Anda di setiap langkah.

Prompt AI Siap Pakai
[PERAN AI] Act as a JavaScript sensei. [TUJUAN] I'm struggling to understand the reduce method. [INSTRUKSI] Explain reduce with a simple analogy (e.g., a factory assembly line). Then, show me three practical examples: 1) Summing an array of numbers, 2) Flattening an array of arrays, 3) Grouping objects in an array by a property.

Proyek Nyata

Proyek 1: Kalkulator Sederhana (Console)

Buat program yang meminta 2 angka dan operator (+, -, *, /), lalu tampilkan hasilnya. Gunakan if/else atau switch untuk memilih operasi.

Proyek 2: Konversi Suhu

Buat fungsi celsiusToFahrenheit(celsius) dan fahrenheitToCelsius(fahrenheit). Panggil fungsi ini dengan beberapa nilai dan tampilkan hasilnya di console.

Proyek 3: To-Do List (Console)

Simpan to-do items dalam sebuah array of objects. Setiap object memiliki id, text, dan completed (boolean). Buat fungsi untuk addTodo, deleteTodo, dan toggleComplete.

Best Practices & Tips

Best Practices untuk Fase 1:

  • Gunakan const secara default, let hanya jika nilai variabel akan berubah. Hindari var.
  • Beri nama variabel yang deskriptif. userName lebih baik dari x.
  • Gunakan === (strict equality) daripada == untuk menghindari bug tak terduga.
  • Fungsi harus melakukan satu hal dengan baik (Single Responsibility Principle).
  • Method map, filter, reduce adalah pilar pemrograman fungsional di JavaScript. Kuasai ini!
  • Jangan mengubah array secara langsung (mutation). Gunakan metode fungsional.
  • Gunakan console.log() secara liberal untuk debugging.
